## Part 2: Create Bug Issue (called from `/bug-intake`)

### Inputs Required

| Input | Source |
|---|---|
| Bug ID | Generated by `/bug-intake` |
| Title | Bug title from triage |
| Severity | P0/P1/P2/P3 |
| Track | A or B |
| Category | From triage classification |
| Module | Current module |
| Pattern match | From 11_bug_patterns.md check |

### Issue Creation Steps

#### Step 1: Build Issue Body

```markdown
## Bug Report: {BUG_ID}

**Module**: {MODULE_NAME}
**Track**: {A (System) / B (Business)}
**Category**: {category}
**Severity**: {P0/P1/P2/P3}
**Pattern Match**: {PAT-XXX / Novel}

### Description
{Bug description from triage}

### Location
- **File**: `{file_path}`
- **Method**: `{method_name}`
- **Line**: {line_number}

### Current Behavior
{What currently happens}

### Expected Behavior
{What should happen}

### Evidence
{DB query results, code snippets}

---
*Auto-generated by Antigravity Bug Remediation System*
*Triage Date: {TODAY}*
*Local Bug ID: {BUG_ID}*
```

#### Step 2: Create the Issue

```
mcp_github-mcp-server_create_issue(
  owner: "{REPO_OWNER}",
  repo: "{REPO_NAME}",
  title: "[{BUG_ID}] {Bug Title}",
  body: "{constructed body}",
  labels: ["{severity_label}", "{track_label}", "{category_label}", "{module_label}", "status:triaged"]
)
```

#### Step 3: Record Issue Number

After creation, GitHub returns the issue number (e.g., `#42`).
- Append `GitHub: #42` to the bug entry in 7_bugs_and_issues.md
- This cross-references local files â†” GitHub

### Output
```

   Title: [{BUG_ID}] {title}
   Labels: {severity}, {track}, {category}, {module}, status:triaged
```

## Part 5: Sync / Reconcile (called from `/overall-bug-dashboard`)

### Step 1: Pull All GitHub Issues

```
mcp_github-mcp-server_list_issues(state: "all", per_page: 100)
```

### Step 2: Cross-Reference

For each bug in 7_bugs_and_issues.md that has a `GitHub: #{N}`:
- Compare local status vs GitHub status
- Flag inconsistencies
- Generate reconciliation report

### Step 3: Handle Offline/Pending

If GitHub is unreachable:
1. Log pending actions to `brain/winbull/_SYSTEM/PENDING_GITHUB_ISSUES.md`:
   ```markdown
   ## Pending Issue: {BUG_ID}
   - Title: [{BUG_ID}] {Bug Title}
   - Labels: {severity}, {track}, {category}, module:{MODULE}
   - Action: CREATE / UPDATE / CLOSE
   - Queued: {TIMESTAMP}
   ```
2. Continue the fix workflow â€” don't block on GitHub
3. On next successful connection, process all pending entries
4. After syncing, archive processed entries

## Commit Message Convention

For cherry-picking and tracking, use this commit format for bug fixes:

```
fix({module}): [{BUG_ID}] {description}

Sprint: {sprint_number}
Track: {A|B}
Severity: {P0|P1|P2|P3}
GitHub: #{issue_number}
```

This makes it trivial to:
- `git log --grep="BUG_ID"` â†’ find the fix commit
- `git log --grep="Sprint: 1"` â†’ find all Sprint 1 fixes
