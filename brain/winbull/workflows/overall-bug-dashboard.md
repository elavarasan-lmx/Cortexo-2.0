# /overall-bug-dashboard Workflow

> Single-view dashboard across ALL Winbull modules — bug status, sprint progress, fix velocity.
> Purpose: Management reporting, sprint reviews, weekly standups.
> Differs from /system-health: This is bug-status focused (what's fixed, what's in progress, who fixed what).

## Usage
Say: `/overall-bug-dashboard`

---

## Prerequisites
- `brain/winbull/artifacts/7_bugs_and_issues.md` is up to date
- Optionally: `brain/winbull/_SYSTEM/FIX_VELOCITY.md` (for velocity trends)

---

## Step 1: Load All Bug Data

Read `brain/winbull/artifacts/7_bugs_and_issues.md`.

Extract per bug:
- Bug ID
- Module
- Severity (P0/P1/P2/P3)
- Category
- Sprint assignment
- Status: ✅ Fixed / ⚠️ Applied (not verified) / 🔧 In Progress / ⏳ Queue / 🔴 Open

Also read `brain/winbull/_SYSTEM/ACTIVE_BUGS.md` if it exists — in-progress/blocked bugs.

---

## Step 2: Calculate Metrics

For each module, compute:
- Total bugs
- Count by severity (P0/P1/P2/P3)
- Count by status (Fixed/Applied/In Progress/Open)
- % Done = (Fixed + Applied) / Total × 100

System-wide:
- Total bugs across all modules
- Overall fix rate %
- Average fix time (if FIX_VELOCITY.md has 5+ entries)
